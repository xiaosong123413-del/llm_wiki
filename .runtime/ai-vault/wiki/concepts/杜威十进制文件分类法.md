# 杜威十进制文件分类法

杜威十进制文件分类法（Dewey Decimal Classification，简称 DDC）是一套以阿拉伯数字为基础的[[分层结构]]分类体系，广泛应用于图书馆馆藏组织，也被个人用户借鉴用于[[文件管理]]与知识整理。其核心思想是：用统一的数字编号将任意领域的内容逐层细分，使分类结构清晰、可扩展、易于检索。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

---

## 基本结构

DDC 依据学科或学问领域来组织内容，将所有知识划分为**十大主分类**，每个主分类下再设 10 个次分类，次分类之下还可通过小数点继续细分。分类编号由 3 位整数构成，分别代表主分类、次分类及含小数点的细项分类。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

十大主分类涵盖哲学、社会科学、自然科学、科技与历史等核心领域。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

### 编号示例

以自然科学与数学为例，层级细分如下：

```
500   自然科学与数学（总论）
510     数学
516       几何
516.3       分析几何
516.37        度量微分几何
516.375         芬斯勒几何
```

每一层数字都遵循相同的层级规则，结构属[[分层结构]]。若不需要非常详细的分类，可缩减最小单位层级的数字，取得更通用的分类。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

## 应用于个人文件管理

DDC 的分层编号思路已被个人用户引入[[文件管理]]实践，尤其适合用文件夹体系管理观点知识类信息。其优势在于：编号排序稳定、层级关系直观、无需依赖特定软件即可实施。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

### 编号前缀与排序

使用数字前缀时，建议从 `01` 而非 `1` 开始编号。原因在于文件系统按字符串排序时，`1` 开头会导致 `11`、`12` 排在 `2` 之前，而 `01`、`02` 则能保证顺序正确。每个文件夹内的条目建议不超过 99 个；若确实超过，则从 `001` 起编。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

### 实践示例

以个人工作文件为例，可按如下方式建立文件夹层级：

```
02 工作/
  01 导演/
  02 配音/
    DUBG_（游戏配音项目）
    DUBR_（广播剧配音项目）
  03 录音/
  04 混音/
  05 声音设计/
```

在次级目录内，可进一步用命名前缀（如 `DUBG`、`DUBR`）替代子文件夹，避免层级过深。建议文件夹嵌套层级不超过六层。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

### 与其他管理方法的关系

DDC 编号法与 [[PARA System]]（Project / Area / Resource / Archive）并不互斥，可以结合使用：PARA 负责按"活跃度"区分内容状态，DDC 编号负责在各分区内部维持有序的学科层级。两者配合，能兼顾动态任务管理与静态知识归档的需求。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

## 优势与局限

**优势**
- 纯数字编号，跨平台、跨软件通用，稳定性高
- 分层结构可按需扩展，细分粒度灵活
- 已在图书馆领域经过长期大规模验证
- 门槛低，无需专用工具，文件夹即可实施

**局限**
- 预设的十大主分类源自图书馆学科体系，直接套用于个人文件时需要自行重新定义分类
- 一个文件只能归入单一编号，难以处理跨领域内容（图书馆通常通过附加标注解决）
- 编号体系一旦建立，大规模重组成本较高
