# 批量剪藏任务池模式

批量剪藏任务池模式是 [[SmartClip]] 提供的一种自动化工作流，核心思路是将待剪藏的网页链接预先汇集到一个"任务池"中，再由 AI Agent 通过 `smartclip-daily` 技能批量处理，无需用户逐条手动触发。该模式目前主要面向移动端的 [[NotionMpClipper]] 和 [[ObClipper]] 用户，后续计划扩展支持各类平台收藏夹等更多批量剪藏场景。^[inbox（剪藏进来的）__notionmpclipper__SmartClip_系列教程之三_给你的_AI_Agent_也配上剪藏插件__4069a2f1.md]

## 核心组件

该模式由以下几个部分协同工作：

- `smartclip-daily`：SmartClip Skills 中专门负责批量处理任务池内容的技能，是该模式的执行入口。
- 任务池：由 [[NotionMpClipper]] 或 [[ObClipper]] 等移动端剪藏工具写入的待处理链接队列。
- [[SmartClip MCP]]：连接浏览器插件与 AI Agent 的通信桥梁，Agent 通过 MCP 调用实际的剪藏能力。
- AI Agent：负责读取任务池、依次调用 `smartclip-daily` 完成剪藏并将结果保存到目标笔记库（如 Notion、Obsidian、思源笔记等）。

^[inbox（剪藏进来的）__notionmpclipper__SmartClip_系列教程之三_给你的_AI_Agent_也配上剪藏插件__4069a2f1.md]

## 与单条剪藏的区别

SmartClip Skills 提供三个主要剪藏技能，定位各有侧重：

| 技能 | 用途 |
|---|---|
| `smartclip-read` | 抓取单条网页内容，供当前上下文使用，不保存 |
| `smartclip-save` | 抓取单条网页内容并保存到笔记库 |
| `smartclip-daily` | 批量处理任务池中的全部待剪藏内容 |

`smartclip-daily` 的价值在于"批量"——用户在移动端积累的待读链接可以一次性交给 Agent 处理，而不需要对每条链接单独发出指令。^[inbox（剪藏进来的）__notionmpclipper__SmartClip_系列教程之三_给你的_AI_Agent_也配上剪藏插件__4069a2f1.md]

## 前置条件

使用该模式前需完成以下配置：

1. 安装 SmartClip 浏览器插件（版本 >= v0.1.2），并在【通用配置】中开启【MCP 服务】，复制 Token 备用。
2. 在 AI Agent（如 Claude Code、Cursor、Gemini CLI 等）的 MCP 配置文件中填入以下配置：

```json
{
  "mcpServers": {
    "smartclip": {
      "command": "npx",
      "args": ["-y", "smartclip-mcp"],
      "env": {
        "SMARTCLIP_MCP_SESSION_TOKEN": "your-secret-token"
      }
    }
  }
}
```

3. 全局安装 SmartClip Skills：

```bash
npm install -g smartclip-agent-extension
smartclip-agent-extension
```

安装完成后，确认技能目录中出现 `smartclip-daily`。^[inbox（剪藏进来的）__notionmpclipper__SmartClip_系列教程之三_给你的_AI_Agent_也配上剪藏插件__4069a2f1.md]

## 注意事项

- MCP 当前绑定 `18365` 端口，且仅支持一对一连接。若 Claude Code 已连接，其他 Agent 客户端将无法同时接入，需关闭其中一个。
- 该功能属于 SmartClip 会员功能，需付费开通后方可使用。
- Agent 剪藏功能发布时处于 beta 状态，可能存在连接不稳定等问题；遇到连接失败时，重启通常可以解决。
- 批量处理会消耗一定量的 AI token，无包月套餐的用户需注意成本。
