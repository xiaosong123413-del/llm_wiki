# 知识库自动化

知识库自动化是指利用 AI 智能体和大语言模型（LLM）接管结构化知识系统中的重复性知识操作，包括维护、交叉链接、查询和编译。这一方法不再将 LLM 单纯视为问答工具，而是将其定位为持续知识工作的主动执行者。

## 核心功能

知识库自动化涵盖四个主要操作层：

- 用于结构化知识查询的 [[Prompt Engineering]]
- 跨编译页面的知识查询与综合
- 知识图谱的自动化维护
- 本地智能体执行与工具协调

### 提示与查询层

有效的自动化查询依赖于结构良好的提示词，需明确指定上下文、任务、格式和约束条件。针对结构化 wiki 的查询比直接查询原始模型更具可追溯性。当查询产生新的结构或结论时，应将这些结果写回知识库，而非丢弃。

### 自动化维护层

维护层负责对知识图谱进行持续的质量控制。自动化例程会检查以下问题：

- 没有入链的孤立页面
- 指向不存在页面的失效链接
- 跨页面的矛盾内容
- 相关概念之间缺失的交叉引用

### 智能体执行层

本地智能体将 LLM 从被动的"回答者"转变为主动的"执行者"。该层将知识库与外部工具、终端和代码仓库相连接，使系统能够基于其知识采取行动，而不仅仅是描述知识。

## 与知识编译的关系

知识库自动化在已编译的知识结构之上运行效果最佳——即原始材料已被处理为稳定、结构化页面的状态。在 [[AI知识库构建]] 模型中，自动化负责处理"摄取 → 编译 → 链接 → 维护"工作流中持续进行的"维护"阶段，使知识图谱长期保持准确且内部一致。

## 自动化任务的规划方法

不同的自动化任务需要匹配不同的 [[Agent Planning Hierarchy]] 层级。简单的文档分析对应 [[Chain-of-Thought]] 推理；动态事实核查和检索任务适合 [[ReAct]]；大规模知识重组或多阶段编译工作流则需要 [[Plan-and-Execute Framework]]。[[Agent Planning Selection Matrix]] 提供了将任务特征与适当规划策略相匹配的具体指导。

## 实用决策指南

| 需求 | 推荐方案 |
|---|---|
| 知识积累与结构化页面 | Wiki 结构（如 Obsidian） |
| 命令执行与工具调用 | 智能体运行时 |
| 移动端访问入口 | 移动智能体入口（如 Litter） |
| 多模型与多渠道治理 | 网关层（如 OpenClaw） |
