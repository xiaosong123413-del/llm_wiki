# 约束工程

约束工程是一种系统性方法，用于在AI工作流程周围构建约束层，旨在创建AI系统无法轻易偏离的框架。它不是简单地提示AI正确行为，而是建立结构性护栏来强制执行期望的行为和结果。

## 核心概念

约束工程的基本原则是，目标不是"提示AI服从"，而是将AI置于一个它无法任意偏离的约束框架中。这种方法认识到，普通提示只能"建议"行为，而经过适当工程设计的约束系统可以"强制执行"行为。

## [[Four-Layer Architecture]]

约束工程将AI工作流程组织成具有四个不同层次的分层系统：

1. **全局指令**（`CLAUDE.md`或等效文件）：定义整体规则和行为风格
2. **记忆**：保存项目上下文、决策和约束
3. **技能**：整合可重用的流程和专门行动
4. **钩子**：在关键节点强制执行必要检查或补充行动

## 钩子的重要性

[[Hooks]]代表约束系统中最关键的组件，因为它们提供了传统提示无法实现的强制执行能力。对于涉及法律、代码或安全考虑的高风险场景，强制执行层比传统提示的建议层更加可靠。

## 在知识管理中的应用

在知识库系统中，约束工程发挥几个保护功能：

- 防止AI直接污染原始数据源
- 强制执行将材料正确吸收到正确目录中
- 在重写页面时保持结构一致性

## 与[[AI Agent Planning]]的关系

约束工程通过为复杂的多步骤任务提供所需的约束框架来补充[[AI-Agent规划]]（[[AI Agent Planning]]）。虽然[[ReAct]]模式处理即时工具使用，[[Plan-and-Execute]]框架管理任务分解，但约束工程确保关键行动在整个执行过程中始终保持在定义的边界内。
