# [[ReAct]] Pattern

[[ReAct]] Pattern 是一种 [[AI Agent]] 推理方式，将思考与行动交替进行。[[ReAct]] 智能体不会孤立地推理，也不会不加反思地行动，而是在生成思路与调用工具之间反复切换，利用每次行动的反馈来决定下一步该做什么。

## 在智能体规划中的位置

智能体规划通常分为三个层次：[[Chain-of-Thought]]（CoT）、[[ReAct]] 和 [[Plan-and-Execute]]。CoT 通过呈现中间步骤来提升局部推理质量。[[ReAct]] 更进一步，将推理与工具使用相结合——智能体先思考，再行动，观察结果，然后再次思考。[[Plan-and-Execute]] 则处于更高层次，在执行开始前就对整个任务进行分解。

## 核心机制

[[ReAct]] 的核心循环如下：

1. 生成推理步骤（"思考"）
2. 选择并调用工具（"行动"）
3. 观察结果（"观察"）
4. 重复上述过程，直至任务完成

这种紧密的反馈循环使智能体能够根据真实的环境响应调整下一步，而无需预先锁定固定计划。

## 适用场景

[[ReAct]] 适合在简单环境中处理短任务。如果任务简短且工具依赖数量较少，[[ReAct]] 通常已经足够。对于依赖项较多或需要分阶段执行的较长任务，则需要像 [[Plan-and-Execute]] 那样进行明确的前期规划。当关键操作需要严格约束时，[[ReAct]] 应与 [[Harness-Engineering]] 配合使用。

## 局限性

能够思考并调用工具，并不意味着智能体就能可靠地完成复杂的多步骤目标。长链任务的失败往往不是推理能力不足造成的，而是缺乏全局规划所致。[[ReAct]] 在设计上是被动响应式的，而非策略性的，这正是它被定位为 [[Plan-and-Execute]] 之下独立层次、而非其替代方案的原因。
