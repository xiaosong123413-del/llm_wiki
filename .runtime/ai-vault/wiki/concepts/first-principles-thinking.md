# 第一性原理思维

> 一种将问题分解至最基本事实、再从零开始重建解决方案的推理方法。

---

## [[Compiled Truth]]

### 核心定义

第一性原理思维是一种问题解决方法，它剥离假设、类比和惯常认知，找出问题背后不可再分的基本事实。在此基础上，解决方案从头构建，而非从现有模型中改造而来。

### 方法步骤

该过程遵循递归追问的模式：

```
问题 → 为什么？（表层原因）
      → 为什么？（更深层原因）
      → 为什么？（再深一层）
      → 为什么？（直到触及一个基本的、不可否认的事实）
      → 从该基本事实出发，重新构建解决方案
```

追问持续进行，直到抵达无法再进一步分解的东西——一个无论惯例或习惯如何都成立的事实。

### 实例解析：个人知识系统

原始资料提供了一个将该方法应用于知识管理的具体示例：

```
问题：为什么现有的笔记应用令人不满意？
  ↓
为什么：因为信息难以快速找到
  ↓
为什么：因为笔记以线性方式存储，搜索仅依赖关键词
  ↓
为什么：因为文件系统是树状结构，而人类思维是网状结构
  ↓
第一性原理：大脑通过关联和情境记忆，而非通过分类记忆
  ↓
重建：设计一个基于链接的系统（双向链接、知识图谱）
  ↓
结果：类似 Obsidian + LLM 知识库的系统
```

这个例子展示了该方法如何从模糊的不满出发，推导出结构性洞见，再落地为具体的设计决策。

### 与其他思维方法的关系

第一性原理思维处于更广泛的推理工具生态之中。原始资料将其与类比思维或惯常思维相对比——后者是对已有事物的改造——并将其与 [[系统思考]]、[[SCAMPER]]、[[二阶思维]] 并列，作为结构化问题解决工具箱的组成部分。[[系统思考]] 问的是"各部分如何相互作用"，而第一性原理问的是"这些部分究竟是什么"。

### 常见误区：将惯例误认为事实

一个关键的失败模式是过早停止"为什么"的追问——将行业规范、习惯性工作流程或既有观点当作基本事实接受，而它们实际上只是一层假设。只有当追问持续到真正不可再分之处，这一方法才能奏效。

### 适用场景

第一性原理思维在以下情况最具价值：

- 现有解决方案令人不满，但原因不明
- 你在设计全新事物，而非在现有基础上迭代
- 惯常方法已无法带来改进
- 你怀疑问题本身的框架方式就是问题的一部分

对于已知现有模式行之有效的日常决策，则无需使用此方法。
