# Concepts Directory Framework

Concepts Directory Framework 是一套知识组织系统，为在更大型知识管理体系中管理可教授的抽象知识划定了清晰的边界与结构。它是 [[三层知识架构]] 的组成部分之一，专门负责组织概念知识层，以区分框架、方法论、理论体系与实际应用、工具及个人项目之间的差异。

## 核心结构

### 目的与范围

概念目录收录所有可教授的抽象知识——思维框架、学习方法、理论体系、专业知识，以及由特定个人创立或倡导的方法论。该目录中的页面回答的是"这是什么，为什么重要"，而非"我如何使用它"。

### 收录标准

该框架明确定义了哪些内容属于概念目录：

- **思维框架**：MECE、第一性原理、[[Systems Thinking]]、设计思维
- **学习方法**：[[Spaced Repetition]]、PBL（问题驱动学习）、费曼学习法
- **理论体系**：信息论、经济学原理、心理学理论
- **专业知识**：法律概念、AI 理论、编程范式
- **文化/哲学体系**：东方哲学、创意方法论
- **个人方法论**：如"Karpathy 的知识库方法"（强调方法论本身，而非人物）

### 排除边界

该框架设立了清晰的边界，以防止分类混乱：

| 排除内容 | 正确位置 | 示例 |
|------------------|----------------|---------|
| 进行中的项目/实践 | `项目/` | "我的备考计划" → 项目/，而非概念/ |
| 软件配置详情 | `工具/` | "Claude 使用指南" → 工具/Claude |
| 原始素材 | `来源/` | 文章摘要、读书笔记 |
| 个人故事 | `人物/` | 某人的生平经历 |
| 工具技术原理 | 可放入 `概念/`，但需与实际应用分开 | "[[ReAct]] 的虚拟 DOM"可放入概念/，具体配置放入工具/[[ReAct]] |

## 实施指南

### 页面模板结构

该框架为概念页面提供了标准化模板，包含以下内容：

- 核心定义与一句话摘要
- [[Compiled Truth]] 部分，包含完整定义、要点与应用场景
- 相关概念及关系说明
- 个人实践应用
- 时间线，用于追踪学习与应用经历

### 消歧规则

该框架针对常见的分类问题给出了解答：

- **方法论 vs. 人物**："Karpathy 的知识库方法"属于概念/，因为关注点在于方法论本身，而非 Andrej Karpathy 这个人
- **归因处理**：著名框架的发明者可在概念页面中提及，无需单独创建人物页面
- **学习笔记**：关于某概念的理解笔记属于概念/，而"我正在上某门课"则属于项目/

## 与知识架构的整合

Concepts Directory Framework 在更宏观的 [[三层知识架构]] 中运作，作为结构化知识层，在原始素材与实际应用之间起到桥梁作用。它与其他目录框架协同工作，共同构建一套完整的知识组织系统，将理论理解与实践应用、个人经验清晰分离。
