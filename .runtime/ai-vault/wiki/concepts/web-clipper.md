# Web Clipper素材捕获

Web Clipper素材捕获是[[Obsidian生产力系统]]中负责将外部网页内容引入知识库的第一道工序。它承担着从开放网络到结构化知识库的入口职责，是整个[[三层知识架构]]中原始来源层（`raw` 层）的主要填充手段。

## 工作流概述

素材捕获的基本流程分为三个环节：网页内容通过 Web Clipper 工具抓取后，先进入收件箱作为临时缓存，再落入 `raw` 层保存为原始证据。`raw` 层的内容遵循只读原则，不直接编辑，确保源材料的完整性与可追溯性。这一流程与[[三层知识架构]]的第1层设计直接对应。

## 常见配置问题

实操中最容易出错的环节是 `vault` 名称和目标目录的相对路径设置。这两项配置直接决定了剪藏内容能否真正落到指定位置——路径配置错误会导致内容散落在错误目录，后续整理成本极高。附件目录如果不预先固定，仓库结构也会很快失控。

## 工具边界

Web Clipper 适合处理网页和文本类内容，不适合用于视频内容的捕获——视频与网页文本是性质不同的输入类型，不应当作同一类素材处理。明确这一边界有助于避免在工作流设计阶段产生错误预期。

## 与知识编译流程的衔接

素材进入 `raw` 层后，后续流程由[[三层知识架构]]的维基层接管：原始素材经过整理形成来源摘要，再通过编译操作合并为正式知识页。Web Clipper 本身只负责捕获，不参与知识的提炼与连线。这种职责分离保证了原始证据的不可变性，同时为后续的知识编译提供了干净的输入。
