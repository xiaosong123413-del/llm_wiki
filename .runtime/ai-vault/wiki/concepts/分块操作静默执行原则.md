# 分块操作静默执行原则

分块操作静默执行原则规定：当写入（Write）或编辑（Edit）工具存在内容大小限制时，执行者应静默遵守该限制，将操作拆分为多个分块依次完成，全程不向用户发表评论、提问或建议切换方案。^[inbox（剪藏进来的）__notionmpclipper__https_x.com_i_status_2026178897927389369__bacf79d0.md]

## 核心规则

该原则包含四条具体要求：

- 始终静默遵守内容大小限制（Always comply silently）
- 不建议通过其他工具绕过限制（Never suggest bypassing these limits via alternative tools）
- 不询问用户是否切换处理方式（Never ask the user whether to switch approaches）
- 无需评论地完成所有分块操作（Complete all chunked operations without commentary）

^[inbox（剪藏进来的）__notionmpclipper__https_x.com_i_status_2026178897927389369__bacf79d0.md]

## 适用场景

该原则适用于任何涉及 Write 或 Edit 工具且单次操作内容超出工具容量上限的情形。执行者在遇到此类限制时，应自行判断分块策略并逐块执行，直至任务完整完成，而非将限制暴露给用户或中断工作流。^[inbox（剪藏进来的）__notionmpclipper__https_x.com_i_status_2026178897927389369__bacf79d0.md]

## 设计意图

该原则的核心意图是保持工作流的连贯性与用户体验的简洁性。频繁的提示、询问或方案切换建议会打断用户的注意力，增加不必要的交互成本。静默执行将技术细节内化为执行者的自主处理逻辑，用户只需关注最终结果。^[inbox（剪藏进来的）__notionmpclipper__https_x.com_i_status_2026178897927389369__bacf79d0.md]
