# PARA 系统

PARA 系统是由生产力专家 Tiago Forte 提出的一套数字信息组织框架，被广泛应用于文件管理、笔记系统和知识管理领域。其核心思想是将所有数字信息归入四个互斥的类别，以行动导向而非主题导向来组织内容。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

## 四个核心类别

PARA 是四个英文单词的首字母缩写：**Project（项目）**、**Area（领域）**、**Resource（资源）**、**Archive（归档）**。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

### Project（项目）

Project 指有明确目标和 Deadline 的事件，是一个能被明确判定是否完成的可量化事件。例如"写一篇关于文件管理的文章"或"做一期视频"。Project 下可以进一步细分为 Task 或 SubProject。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

### Area（领域）

Area 是需要长期关注、学习、维护的领域，或者说是宏观目标，没有明确的完成标志和完成时间。例如"学习厨艺"是 Area，而"学做咸蛋黄鸡翅"则是由此衍生的 Project。Area 会不断产生新的 Project 来推进。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

### Resource（资源）

Resource 是外部或内部产生的知识点，作为 Area 和 Project 的知识储备与支撑，需要时进行调用。例如与文件管理相关的论文、博主视频、随笔灵感等，都归入 Resource。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

### Archive（归档）

Archive 是已完成、不再需要主动关注但尚不能删除的内容。例如某个项目完结后，将相关文件整合、清理冗余内容后封存至 Archive，以备后续 Recall 或作为案例参考。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

## 类别之间的关系

Project 与 Area 的区别在于是否有明确的完成节点：通过推进一个个 Project，可以持续推动 Area 的发展；Resource 为 Project 和 Area 提供素材支撑；Archive 则是 Project 完成后的最终去向。这四个类别覆盖了信息从活跃使用到沉淀封存的完整生命周期。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

## 实施方式

PARA 系统与具体工具无关，可以在文件夹、[[Obsidian生产力系统|Obsidian]]、Notion 或任何笔记软件中实现。最直接的做法是在根目录下建立 `Project`、`Area`、`Resource`、`Archive` 四个文件夹，在 Project 文件夹内用时间前缀（如 `2601_学做咸蛋黄鸡翅`）标注项目，便于排序和追踪。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

## 归档的意义

及时将完成的项目移入 Archive，不仅是释放存储空间，更重要的是从精力层面解放使用者——明确的归档动作如同划掉任务清单上的一项，给人以仪式感和正反馈，让注意力得以从已完成的事务中真正抽离。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]

## 局限性

PARA 系统并非适合所有人。其行动导向的分类逻辑对于以主题或学科为核心的知识管理场景（如学术研究、百科式知识库）可能不够自然。实践者需根据自身工作习惯和信息类型酌情调整，不必照单全收。^[inbox（剪藏进来的）__notionmpclipper__2026_关于文件管理的思考_-_少数派__e7626f05.md]
