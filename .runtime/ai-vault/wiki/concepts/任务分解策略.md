# 任务分解策略

## 概述

**任务分解策略** 指在开始执行前，先把大任务拆成若干边界清晰、验证条件明确、提示词自包含的小任务。当前知识库里，这一步被视为长程任务成败的关键。

## 在当前知识库中的作用

相关材料强调，子任务不能写成“根据之前分析继续做”，而应写清文件范围、错误现象和预期行为。这样子 agent 才能在独立上下文里稳定工作，而不会依赖父会话历史。
