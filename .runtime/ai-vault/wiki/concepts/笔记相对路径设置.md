# 笔记相对路径设置

笔记相对路径设置是在使用Obsidian Web Clipper等工具时，指定笔记保存位置的重要配置项。正确设置相对路径可以确保剪藏的内容保存到预期的文件夹位置，而不是散落在知识库根目录中。

## 基本概念

相对路径是指从知识库根目录开始的文件路径，不包含知识库名称本身。在Obsidian中，所有的文件操作都基于当前打开的vault（保管库）进行，因此路径设置需要遵循vault内部的文件夹结构。

## 常见问题与解决方案

### Vault Not Found 错误

在配置Obsidian Web Clipper时，经常出现"vault not found"错误。这个问题的根本原因是保管库名称填写错误。vault字段需要填写的是你的知识库的实际名称，而不是路径或其他标识符。

### 笔记位置配置

剪藏工具无法将内容保存到指定位置的主要原因是笔记位置配置不正确。在笔记位置字段中，需要填写的是笔记库的相对路径，而不是绝对路径或包含vault名称的完整路径。

## 配置示例

假设你的知识库结构如下：
```
我的知识库/
├── 01_收集箱/
├── 02_领域/
│   └── obsidian/
│       └── 剪藏/
└── 03_项目/
```

如果要将剪藏内容保存到"02_领域/obsidian/剪藏/"文件夹，相对路径应该设置为：
```
02_领域/obsidian/剪藏/
```

## 跨平台注意事项

在配置移动端Obsidian时，相对路径设置同样适用。无论是桌面端还是移动端，路径格式保持一致，确保跨设备同步时文件位置的准确性。

## 最佳实践

1. **使用标准化文件夹结构**：建立清晰的文件夹层次，便于设置和维护相对路径
2. **避免特殊字符**：文件夹名称尽量使用字母、数字和下划线，避免空格和特殊符号
3. **测试路径有效性**：配置完成后进行测试，确保文件能够正确保存到指定位置
4. **保持路径简洁**：过深的文件夹层次会增加路径配置的复杂性
