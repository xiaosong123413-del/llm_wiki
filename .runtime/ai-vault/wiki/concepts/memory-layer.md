# Memory Layer

Memory Layer 是 [[三层知识架构]] ([[Three-Tier Knowledge Architecture]]) 中的第二层，是 AI 知识库系统中动态演化的核心。与只读的原始素材层不同，Memory Layer 会随着新信息的处理与整合持续生长，并将其纳入结构化的知识页面。

## 结构与组织

Memory Layer 对应 `ai知识库（第二大脑）` 目录结构，细分为两个主要组成部分：

### 来源管理（`来源/`）
该子目录存储来源摘要与吸收记录，维护原始素材与已处理知识之间的可追溯性。它是一个中间处理区，原始材料在此初步消化后，再汇编为正式的知识页面。

### 知识分类
Memory Layer 将正式知识内容组织为不同类别：
- `概念/` - 核心概念与理论框架
- `人物/` - 个人信息及其贡献
- `工具/` - 工具、技术与方法论
- `项目/` - 项目文档与案例研究
- `想法/` - 创意、洞见与灵感
- `写作/` - 书面内容与文档

这种分类方式在支持相关概念跨引用链接的同时，实现了系统化的知识组织。

## 核心功能

### 知识汇编
Memory Layer 是汇编流程的主要目的地，多个来源材料在此被综合为连贯、结构化的知识页面。这一汇编过程将原始层中碎片化的信息转化为全面、相互关联的知识资产。

### 持续演化
与静态文档系统不同，Memory Layer 专为持续精炼而设计。随着新信息的涌入与理解的深化，知识页面会定期更新、扩充并相互关联。这种演化式方法确保知识库始终保持时效性，并随时间推移愈发有价值。

### 查询优化
长期查询应优先指向 Memory Layer，而非直接访问原始素材。这一方式充分利用了 Memory Layer 内容结构化、经过处理的特性，从而提供更连贯、上下文更丰富的响应。

## 与其他层的关系

### 与原始层的整合
Memory Layer 通过吸收流程接收来自原始素材层的输入，但保持严格隔离以保护原始来源的完整性。这种单向流动确保已处理的知识始终可追溯至其来源，同时防止对原始素材造成污染。

### 对配置层的依赖
Memory Layer 依据配置层（`RESOLVER.md`、`schema.md`、`CLAUDE.md`）中定义的规则与标准运行。这些配置文件确保 Memory Layer 所有内容在命名规范、结构组织与汇编流程上保持一致。

## 在 Obsidian 中的实现

在 [[Obsidian生产力系统]] 中，Memory Layer 充分利用 Obsidian 的核心功能进行知识管理：

- **双向链接** 通过 wikilink 连接构建知识网络
- **图谱可视化** 提供知识关系洞察，并识别潜在的知识空白
- **模板系统** 确保知识页面格式与结构的一致性
- **标签层级** 支持超越文件夹分类的多种组织方案

Memory Layer 的结构与 Obsidian 在处理相互关联、持续演化的知识方面的优势高度契合，同时保持清晰的组织原则。

## 约束执行

当与 [[Harness Engineering]] 原则整合时，Memory Layer 受益于结构性约束，这些约束能防止知识质量退化。它们确保知识汇编遵循既定模式，并在更新与重组过程中保留关键信息。
