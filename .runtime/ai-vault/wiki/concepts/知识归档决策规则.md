# 知识归档决策规则

归档决策规则定义了在[[Obsidian生产力系统]]中，何时将某个页面、项目或内容从活跃状态转入归档状态的判断标准。其目的是保持知识库的整洁与可用性，同时避免过早或过晚归档导致的信息丢失或噪音积累。

## 归档触发条件

归档操作需满足以下任一条件：

- 项目已完成，且近期不再推进
- 主题长期无变化，且不再影响当前判断
- 临时捕获卡已回收到日记，不再需要独立存在

三类条件分别对应项目类页面、主题类页面和临时捕获类内容，覆盖了知识库中主要的页面类型。

## 与新建页面规则的对应关系

归档决策与新建页面的判断逻辑互为镜像。新建项目页的条件是"有明确终点、完成后可结束、需要持续推进"；一旦这些条件不再成立——即项目已完成且不再推进——归档条件即被触发。新建主题页的条件是"不是一次性事务、未来还会继续补充更新、值得长期维护"；当主题长期无变化且不再影响判断时，同样应当归档。

## 稳定认知的升级门槛

内容在归档之前，可能经历从临时捕获到稳定认知的升级过程。将内容升为稳定认知需同时满足三个条件：已进入"最近变化区"；从第一次出现起已观察满 1 个月；期间没有出现相反证据。这一机制确保只有经过时间验证的内容才会被固化，而非仓促归档或升级。

## 项目日志的归档时机

[[项目日志]]并非持续写入，而是仅在以下内容发生变化时才记录：项目目标变了、项目阶段变了、核心问题变了、主下一步动作变了。这意味着日志条目的停止新增，本身就是项目趋于稳定或完成的信号，可作为判断是否归档的辅助依据。

## 与 Agent 集成工作流的关系

在[[Agent集成工作流]]中，归档决策规则可作为 Agent 执行自动化维护的判断依据。Agent 负责识别符合归档条件的页面并执行结构维护，人则专注于价值判断——即确认某个项目或主题是否真正满足归档条件，而非手动执行归档操作本身。
