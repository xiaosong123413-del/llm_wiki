# 项目页面模板标准

项目页面模板标准定义了在知识管理系统中创建和维护项目页面的统一格式和结构要求。该标准确保所有正在进行的工作项目都能以一致的方式记录、跟踪和管理，支持从项目启动到完成的全生命周期管理。

## 适用范围

### 包含的项目类型

项目页面模板适用于所有有明确owner、正在推进的工作项目或学习计划：

- **工作任务**：考研备考、论文写作、课程学习
- **个人创业项目**：产品开发、服务创意
- **学习计划**：系统学习AI、掌握某个技能
- **产品开发**：从规划到发布的产品项目
- **团队协作**：部门项目、团队任务

### 排除的内容类型

以下内容不适用项目页面模板，应使用其他分类：

| 不包括 | 放置位置 | 示例 |
|--------|---------|------|
| 尚未开始的想法 | `想法/` | "应该学Rust"但还没开始 |
| 已完成的项目 | `归档/` | 2025考研备考已完成 |
| 纯理论研究 | `概念/` | "法律考试复习方法论" |
| 工具使用教程 | `工具/` | "如何用Claude" |
| 单篇文章 | `写作/` | 待发布的一篇博客文章 |

## 标准模板结构

### 页面头部

每个项目页面以项目名称作为标题，并包含一句话项目目标描述：

```markdown
# [项目名称]

> [一句话项目目标]
```

### Compiled Truth 部分

这一部分包含项目的核心信息，分为四个子模块：

**项目定义**
- **Owner**：项目负责人
- **时间框**：项目时间范围（如2026-04-11 ~ 2026-12-31）
- **状态**：Planned/In Progress/On Hold/Completed

**目标**
- 终极目标：定量的最终目标
- 中间里程碑：带日期的阶段性目标

**当前进展**
- 完成百分比：项目完成度
- 最近更新：最新进展描述
- 下一步行动：具体的下一步任务

**关键资源**
- 所用工具列表
- 相关概念链接

### Timeline 部分

Timeline部分按时间顺序记录项目的重要节点和进展更新，格式为：

```markdown
### YYYY-MM-DD - 事件标题
- 完成内容描述
- 后续计划说明
```

## 使用指南

### 项目识别标准

判断某项工作是否应创建项目页面的关键标准：
- 有明确的owner（负责人）
- 有明确的开始和结束时间
- 有可衡量的中间里程碑
- 正在积极推进中

### 生命周期管理

项目页面在不同阶段的处理方式：
- **进行中**：保持在项目目录，定期更新进展
- **已完成**：移动到归档目录，保留所有历史记录用于学习回顾
- **暂停**：状态标记为"On Hold"，说明暂停原因

## 常见问题

**学习课程是否算作项目？**
如果有明确的owner、开始和结束时间、中间里程碑，就是项目。否则属于想法分类。

**人生目标是否算作项目？**
通常不是。除非正在为之制定具体的、可衡量的行动计划，否则属于想法分类。

**已完成的项目是否保留在项目目录？**
不保留。完成后移到归档目录，但保留所有历史记录用于学习回顾。
