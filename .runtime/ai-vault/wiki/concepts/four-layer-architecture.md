# 四层架构

四层架构是一种结构化方法，用于组织AI代理系统和知识管理工作流，它扩展了传统的三层模型，增加了额外的约束执行层。这种架构为管理复杂的AI驱动流程提供了全面的框架，同时保持系统可靠性和行为一致性。

## 四个层次

### 第一层：全局指令
基础层由系统范围的规则和行为准则组成，通常通过`CLAUDE.md`等文件或全局指令集实现。这一层定义了整体操作参数、沟通风格和基本约束，这些约束管理系统内所有AI交互。

### 第二层：记忆
[[Memory Layer]]保存项目上下文、决策历史和累积约束。这种持久存储确保跨会话的连续性，并维护关于过去决策、成功模式和应该影响未来操作的学习约束的机构知识。

### 第三层：技能
技能层包含通过实践精炼的可重用流程和专门行动。这些代表了提炼的专业知识和经过验证的工作流，可以在类似场景中一致应用，减少了为常见任务重新发明方法的需要。

### 第四层：钩子
钩子层提供在关键决策点运行的执行机制。与主要建议或指导行为的其他层不同，钩子可以强制执行特定行动或验证。这一层对于涉及法律合规、安全要求或代码安全的高风险场景至关重要，在这些场景中仅有建议是不够的。

## 与提示的区别

四层架构代表了从传统提示方法的根本转变。这种架构不是仅仅依靠自然语言建议来指导AI行为，而是创建AI无法轻易偏离的结构性约束。目标不是"提示AI表现"，而是将AI置于强制执行期望行为的约束框架内。

## 关键应用

这种架构在行为可靠性至关重要的场景中特别有价值：

- **法律工作流**：确保符合监管要求并防止未经授权的行动
- **代码生成**：维护安全标准并防止危险操作
- **知识库管理**：保护数据完整性并执行组织结构
- **安全关键系统**：保证无论上下文如何都永远不会跨越某些边界

## 在知识系统中的实现

在知识管理环境中，四层架构服务于特定的保护功能：

- 防止AI直接修改受保护的内容区域，如`raw`目录
- 强制执行新材料的正确分类和路由
- 在内容更新和重组期间维护结构一致性
- 确保知识编译遵循既定标准和格式

## 与规划框架的关系

四层架构通过提供可靠执行所需的约束基础设施来补充[[AI Agent Planning]]框架。虽然像[[Plan-and-Execute Framework]]这样的规划框架专注于任务分解和排序，但四层架构确保执行在整个过程中保持在可接受的边界内。
