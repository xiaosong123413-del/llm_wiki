# 对话页双形态设计

## 目标

将 `LLM Wiki` 的对话页从当前的“编译工作台”改为“聊天优先”的知识库工作区，并支持两种可切换形态：

1. 纯对话态：不显示页面预览栏，聊天区最大化。
2. 预览展开态：聊天区仍然可见，同时显示右侧页面预览栏。

本次设计只重构对话页，不重写审查、设置、日志页面的核心交互逻辑。

## 设计原则

- 对话页优先服务“提问、连续对话、附加上下文”。
- 文件浏览、会话管理、聊天回复、页面预览必须边界清晰。
- 预览不是常驻栏，而是按需展开。
- 用户拖拽修改过的栏宽要被记住。
- 聊天页与同步、检查、设置、日志页面分工明确，不再混用“编译控制台”风格。

## 用户体验

### 默认进入

- 用户点击左侧“对话”后，进入纯对话态。
- 页面显示四块区域：
  - 全局导航栏
  - 文件浏览栏
  - 会话栏
  - 聊天主区

### 打开预览

- 用户在文件浏览栏中双击某个文件。
- 对话页切换到预览展开态。
- 聊天区保留，右侧新增页面预览栏。
- 当前预览文件名显示在预览栏顶部。

### 关闭预览

- 用户点击预览栏关闭按钮。
- 对话页回到纯对话态。
- 聊天区自动扩展回剩余宽度。

### 上下文附加

- 选中模式下，单击文件不会打开预览，而是把文件加入聊天上下文。
- 普通模式下，双击文件才打开预览。
- 预览中的当前文件默认视为当前轮对话的可用上下文来源之一。

## 页面结构

### 纯对话态

从左到右：

1. 全局导航栏
2. 文件浏览栏
3. 会话栏
4. 聊天主区

### 预览展开态

从左到右：

1. 全局导航栏
2. 文件浏览栏
3. 会话栏
4. 聊天区
5. 页面预览栏

说明：

- “预览展开态”不是跳转新页面，而是在对话页内部展开右栏。
- 聊天区和预览区必须同时可见。

## 组件划分

### 1. 全局导航栏

固定按钮顺序：

- 对话
- 检查
- 同步
- 审查
- 日志
- 设置

职责：

- 只负责全局页面切换或动作入口。
- 不承载文件状态或会话状态。

### 2. 文件浏览栏

包含：

- `wiki / raw层` tab
- 搜索框
- `+` 按钮
- `筛选` 按钮
- 文件树
- `选中模式`

职责：

- 浏览文件结构
- 搜索文件
- 切换知识层
- 在选中模式下把文件加入上下文
- 在普通模式下通过双击打开预览

### 3. 会话栏

包含：

- `+ New Chat`
- 会话列表
- 空状态文案

职责：

- 创建会话
- 切换会话
- 展示会话标题和最近使用状态

### 4. 聊天主区

包含：

- 会话标题区
- 消息流
- 空状态插画与文案
- 输入框
- 发送按钮
- 当前上下文提示区

职责：

- 渲染用户与 AI 的消息
- 接收输入
- 展示本轮上下文来源
- 在纯对话态与预览展开态下都保持可用

### 5. 页面预览栏

仅在预览展开态显示，包含：

- 标题：`页面`
- 当前文件名
- 关闭按钮
- Markdown 预览内容

职责：

- 展示当前双击打开的文件内容
- 不承担编辑职责
- 不替代聊天区

## 栏宽与拖拽

### 要求

所有主栏宽度均支持手动拖拽调节。

### 纯对话态可调分隔

- 导航栏 / 文件栏
- 文件栏 / 会话栏
- 会话栏 / 聊天区

### 预览展开态可调分隔

- 导航栏 / 文件栏
- 文件栏 / 会话栏
- 会话栏 / 聊天区
- 聊天区 / 预览区

### 默认宽度

- 导航栏：`76px`
- 文件栏：`270px`
- 会话栏：`320px`
- 纯对话态聊天区：剩余宽度
- 预览展开态：
  - 聊天区：剩余主区的 `45%`
  - 预览区：剩余主区的 `55%`

### 最小宽度

- 导航栏：`76px`
- 文件栏：`220px`
- 会话栏：`260px`
- 聊天区：`360px`
- 预览区：`320px`

### 宽度持久化

需要把用户拖拽后的宽度写入本地配置，至少包括：

- `nav_width`
- `file_browser_width`
- `conversation_width`
- `chat_width_with_preview`
- `preview_width`

规则：

- 重启应用后恢复上次宽度
- 关闭预览只隐藏预览栏，不丢弃预览态宽度
- 再次打开预览时恢复用户上次的聊天区/预览区比例

## 事件规则

### 文件交互

- 单击文件：选中该文件
- 双击文件：打开预览栏
- 选中模式下单击文件：加入上下文，不打开预览

### 会话交互

- 点击 `+ New Chat`：创建新会话并清空消息流显示
- 切换会话：恢复该会话消息记录
- 切换会话时默认不强制关闭预览，但预览文件与会话状态分离管理

### 发送消息

- 发送消息时，聊天区消息流追加用户消息
- AI 响应以消息块形式追加
- 当前预览文件和已选中文件应作为本轮上下文输入的一部分

## 状态模型

对话页至少维护以下状态：

- 当前页面是否为对话页
- 当前是否处于预览展开态
- 当前 `wiki/raw层`
- 当前搜索关键词
- 当前选中文件
- 当前预览文件
- 当前是否处于选中模式
- 当前已附加上下文文件列表
- 当前会话 ID
- 会话消息列表
- 分栏宽度配置

## 技术方案

### 推荐实现

使用 `WinForms SplitContainer` 组合实现多栏拖拽，而不是手动用固定 `TableLayoutPanel.ColumnStyles.Width` 模拟拖拽。

建议结构：

- 外层主框架仍保留现有导航栏 + 文件栏 + 内容区的总容器
- 文件栏与内容区：一层 `SplitContainer`
- 会话栏与主工作区：一层 `SplitContainer`
- 预览展开态时，聊天区与预览区：再嵌套一层 `SplitContainer`

这样可以直接获得：

- 原生拖拽分隔
- 更清晰的宽度管理
- 更稳定的展开/收起行为

### 当前代码的迁移方向

- 保留全局导航栏
- 保留文件浏览栏主体结构
- 重写 `chatViewPanel`
- 将当前 `queryResultTextBox` 替换为消息流视图
- 将当前 `queryTextBox` 移动到聊天区底部输入栏
- 新增预览栏容器与展开/关闭逻辑

## 错误处理

- 预览文件不存在：预览栏显示“文件不存在或已被移除”
- Markdown 读取失败：预览栏显示读取错误提示，不影响聊天区
- 宽度配置损坏：回退到默认宽度
- 会话状态缺失：回退到空会话
- 当前预览文件失效：自动关闭预览栏或显示空预览提示

## 测试要求

至少覆盖以下验证点：

1. 对话页默认是纯对话态
2. 文件双击后切换到预览展开态
3. 预览关闭后恢复纯对话态
4. 审查、设置、日志页面不复用对话页文件栏逻辑
5. 日志页仍保留独立页面入口
6. 栏宽配置能保存并恢复
7. 选中模式下单击文件不会误打开预览
8. 普通模式下双击文件能打开预览

## 实施顺序

1. 重构对话页静态布局
2. 引入会话栏与聊天区基础容器
3. 实现纯对话态
4. 实现文件双击打开预览
5. 实现预览关闭返回纯对话态
6. 实现拖拽分栏与宽度持久化
7. 将查询输出接入消息流
8. 接入“当前预览文件 + 已选中文件”的上下文逻辑

## 范围边界

本次不包含以下内容：

- 富文本编辑
- 页面内直接修改并保存
- 多标签预览
- 审查页与设置页的全面视觉重构
- 网络搜索 API 的实际接入
- 向量检索的实际接入

## 结论

对话页应被重构为一个支持“纯对话态 / 预览展开态”的聊天优先工作区。

默认以纯对话态进入；双击文件时进入预览展开态；聊天区始终保留；所有主栏宽度允许用户拖拽并持久化保存。

这是当前 WinForms 技术栈下最稳妥、最接近目标视觉、也最便于后续继续演进的方案。
